Fundamentals of Hypothesis Testing

In Simple Language

Hypothesis testing decides between two competing claims about a parameter. The null H₀ is the status quo (e.g. μ = μ₀); the alternative H₁ is what we want to detect (μ ≠, < or > μ₀). We compute a test statistic (Z or t) and either compare it with a critical value or compare its p-value with α. Reject H₀ if |test stat| > critical value, or if p < α. Type I error (α) — rejecting a true H₀ (false alarm). Type II error (β) — failing to reject a false H₀ (missed detection). Power = 1 − β. For a mean: Z = (x̄ − μ₀)/(σ/√n) if σ known, else t = (x̄ − μ₀)/(s/√n). For a proportion: Z = (p̂ − p₀)/√[p₀(1−p₀)/n].

Key Concepts

  • H₀ (null) vs H₁ (alternative); one-tail vs two-tail
  • Significance level α and p-value
  • Test statistic for mean (Z, t) and proportion (Z)
  • Critical-value approach vs p-value approach
  • Type I error (α) and Type II error (β); Power = 1 − β
  • Statistical vs practical significance

Quiz Practice Quiz

Q1. Rejecting H₀ when it is actually true is:
  • Type I error
  • Type II error
  • Correct decision
  • Power
Explanation: Type I = false alarm, probability α.
Q2. If p-value = 0.03 and α = 0.05:
  • Fail to reject H₀
  • Reject H₀
  • Need more data
  • Accept H₀
Explanation: p < α ⇒ reject H₀; evidence against the null is strong enough.
Q3. For H₁: μ > 100 the test is:
  • Two-tailed
  • Right-tailed
  • Left-tailed
  • Chi-square
Explanation: Inequality points to the right tail.
Q4. Power of a test is:
  • α
  • β
  • 1 − α
  • 1 − β
Explanation: Power = probability of correctly rejecting a false H₀.

Short Answers Short-Answer Q&A

Difference between Type I and Type II errors.
Type I (α): rejecting a true H₀. Type II (β): failing to reject a false H₀. Reducing α tends to increase β; only larger n can reduce both.
What is a p-value?
The probability of observing a test statistic at least as extreme as the one obtained, assuming H₀ is true; small p-values are evidence against H₀.
Define statistical significance.
A result is statistically significant at level α if the p-value is less than α; the evidence is strong enough to reject H₀.
List the 5 steps of hypothesis testing.
(1) State H₀ and H₁; (2) Choose α; (3) Identify the test statistic and its distribution; (4) Compute the statistic and p-value (or critical region); (5) Decide and interpret in business terms.

Long Answer 150-Word Answer

Hypothesis testing is a structured framework for deciding between two competing claims about a population parameter using sample evidence. We begin by formulating the null hypothesis H₀ — the conservative, status-quo statement we will only abandon with strong evidence — and the alternative H₁, which captures the effect we are trying to demonstrate. The test can be two-tailed (≠) or one-tailed (>, <). We then choose a significance level α, commonly 0.05, which fixes our tolerance for a Type I error (rejecting a true H₀). We compute a test statistic — Z for known σ or large samples, t for unknown σ with df = n − 1 — that measures how many standard errors the sample estimate lies from the hypothesised value. The decision can be made by the critical-value approach (reject H₀ if the statistic falls in the rejection region) or the equivalent p-value approach (reject H₀ if p < α). Two errors are possible: Type I with probability α, and Type II (failing to reject a false H₀) with probability β; the test's power, 1 − β, is its ability to detect a real effect. Lowering α tends to raise β; only a larger sample can reduce both. Crucially, a statistically significant result is not necessarily practically important — with a huge n even tiny, business-irrelevant effects become significant, so managers must always interpret in context.
